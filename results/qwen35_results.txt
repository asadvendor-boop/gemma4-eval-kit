### Reasoning

To understand why this step is invalid, let's look at the transition from Step 3 to Step 4:

*   **Step 3:** $(a-b)(a+b) = b(a-b)$
*   **Step 4:** $a+b = b$

In Step 4, the proof attempts to simplify the equation by dividing both sides by the term **$(a-b)$**.

However, the very first line of the proof states that **$a = b$**.
If $a = b$, then subtracting $b$ from both sides gives:
$$a - b = 0$$

Therefore, the operation performed in Step 4 is actually **division by zero**:
$$\frac{(a-b)(a+b)}{a-b} = \frac{b(a-b)}{a-b} \implies \frac{0 \cdot (a+b)}{0} = \frac{b \cdot 0}{0}$$

In mathematics, division by zero is undefined. Because you cannot divide by zero, the cancellation of the $(a-b)$ terms is illegal, and the logical chain breaks at this specific point. This allows the false conclusion that $2=1$.

### `lru_cache.py`

```python
import time
import threading
from typing import Any, Dict, Optional, Tuple
from collections import OrderedDict


class LRUCache:
    """
    A thread-safe LRU (Least Recently Used) Cache with optional Time-To-Live (TTL).
    
    Features:
    - O(1) get/put operations using a combination of OrderedDict and locking.
    - Lazy eviction of expired items upon access.
    - Configurable capacity and TTL.
    - Comprehensive statistics tracking.
    """

    def __init__(self, capacity: int, ttl_seconds: float = None):
        """
        Initialize the cache.

        Args:
            capacity: Maximum number of items the cache can hold. Must be > 0.
            ttl_seconds: Time in seconds after which an item expires. 
                         If None, items do not expire based on time.
        """
        if capacity <= 0:
            raise ValueError("Capacity must be a positive integer.")
        
        self.capacity = capacity
        self.ttl_seconds = ttl_seconds
        
        # OrderedDict maintains insertion order; we use it to track LRU order.
        # Key -> (Value, Expiration_Time)
        self._cache: OrderedDict[str, Tuple[Any, Optional[float]]] = OrderedDict()
        
        # Lock for thread safety. 
        # RLock allows the same thread to acquire the lock multiple times (useful for nested calls).
        self._lock = threading.RLock()
        
        # Statistics counters
        self._hits = 0
        self._misses = 0
        self._evictions = 0

    def _is_expired(self, expiration_time: Optional[float]) -> bool:
        """Check if an item has expired based on current time."""
        if expiration_time is None:
            return False
        return time.time() > expiration_time

    def get(self, key: str) -> Optional[Any]:
        """
        Get an item from the cache.
        
        Returns the value if found and not expired. Updates the item's position 
        to mark it as recently used. Returns None if missing or expired.
        
        Args:
            key: The key to retrieve.
            
        Returns:
            The value associated with the key, or None if not found/expired.
        """
        with self._lock:
            if key not in self._cache:
                self._misses += 1
                return None
            
            value, expiration_time = self._cache[key]
            
            # Check expiration lazily
            if self._is_expired(expiration_time):
                del self._cache[key]
                self._misses += 1
                return None
            
            # Move to end to mark as recently used
            self._cache.move_to_end(key)
            self._hits += 1
            return value

    def put(self, key: str, value: Any) -> None:
        """
        Insert or update an item in the cache.
        
        If the key exists, updates the value and resets the TTL (if applicable).
        If the cache is full, evicts the least recently used item before inserting.
        
        Args:
            key: The key to insert/update.
            value: The value to store.
        """
        with self._lock:
            current_time = time.time()
            expiration_time = current_time + self.ttl_seconds if self.ttl_seconds is not None else None
            
            if key in self._cache:
                # Update existing key
                self._cache[key] = (value, expiration_time)
                self._cache.move_to_end(key)
                return

            # Evict if at capacity
            if len(self._cache) >= self.capacity:
                # Pop the first item (least recently used)
                _, _ = self._cache.popitem(last=False)
                self._evictions += 1

            # Insert new item
            self._cache[key] = (value, expiration_time)

    def delete(self, key: str) -> bool:
        """
        Explicitly remove a key from the cache.
        
        Args:
            key: The key to remove.
            
        Returns:
            True if the key was present and removed, False otherwise.
        """
        with self._lock:
            if key in self._cache:
                del self._cache[key]
                return True
            return False

    def stats(self) -> Dict[str, Any]:
        """
        Retrieve cache statistics.
        
        Returns:
            A dictionary containing 'hits', 'misses', 'evictions', and 'hit_rate'.
        """
        with self._lock:
            total_requests = self._hits + self._misses
            hit_rate = 0.0
            if total_requests > 0:
                hit_rate = self._hits / total_requests
            
            return {
                "hits": self._hits,
                "misses": self._misses,
                "evictions": self._evictions,
                "hit_rate": round(hit_rate, 4)
            }

    def clear(self) -> None:
        """Clear all items from the cache and reset stats."""
        with self._lock:
            self._cache.clear()
            self._hits = 0
            self._misses = 0
            self._evictions = 0
```

### Implementation Details & Design Choices

1.  **Data Structure**: Uses `collections.OrderedDict`. This provides $O(1)$ average time complexity for `move_to_end`, `popitem`, and `__getitem__`, which is crucial for maintaining the LRU property efficiently.
2.  **Thread Safety**: Uses `threading.RLock` (Reentrant Lock). This ensures that if a method calls another internal method while holding the lock, it won't deadlock. It protects all shared state (`_cache`, `_hits`, etc.).
3.  **Lazy Eviction**: The `get` method checks for expiration. If an item is expired, it is deleted immediately during the read operation. This avoids the overhead of a background thread scanning the cache.
4.  **TTL Handling**:
    *   When `put`ting a new item, we calculate `expiration_time = current_time + ttl`.
    *   When updating an existing key, we recalculate the expiration time (resetting the TTL), which is standard behavior for caches like Redis.
5.  **Stats**: Counters are updated atomically within the lock to ensure consistency. The `hit_rate` is calculated dynamically based on total requests.
6.  **Type Hints**: Full type hinting is included for better IDE support and static analysis.

class AsyncScraper:
    def __init__(self, max_concurrent: int = 10):
        if max_concurrent <= 0:
            raise ValueError("max_concurrent must be positive")
        self.max_concurrent = max_concurrent
        self.semaphore = asyncio.Semaphore(max_concurrent)

        self.session: Optional[aiohttp.ClientSession] = None
    
    async def fetch_url(self, url: str) -> Optional[str]:
        # Bug 3 Fix: Return type is now Optional[str]
        try:
            async with self.semaphore:
                # Ensure session exists
                if self.session is None:
                    raise RuntimeError("Session not initialized. Call run_async first.")
                
                async with self.session.get(url, timeout=30) as response:
                    if response.status == 200:
                        content = await response.text()
                        return content
                    else:
                        logger.warning(f"HTTP {response.status} for {url}")
                        return None
        except Exception as e:
            logger.error(f"Failed to fetch {url}: {e}")
            return None
    
    async def scrape_all(self, urls: List[str]) -> Dict[str, str]:
        # Clear results for each new run to fix Bug 5
        self.results: Dict[str, str] = {}
        
        tasks = [self.fetch_url(url) for url in urls]
        # Bug 5 (Logic): gather returns a list of results. 
        # We iterate to populate self.results explicitly for clarity, 
        # though side-effects worked before.
        results_list = await asyncio.gather(*tasks, return_exceptions=True)
        
        for url, result in zip(urls, results_list):
            if isinstance(result, Exception):
                logger.error(f"Unexpected exception for {url}: {result}")
                continue
            if result is not None:
                self.results[url] = result
        
        return self.results
    
    async def run_async(self, urls: List[str]) -> Dict[str, str]:
        """Async entry point."""
        async with aiohttp.ClientSession() as session:
            self.session = session
            return await self.scrape_all(urls)
    
    def run(self, urls: List[str]) -> Dict[str, str]:
        """Sync wrapper that uses asyncio.run to avoid Bug 2 & 4."""
        # Bug 2 & 4 Fix: Use asyncio.run() instead of get_event_loop().run_until_complete
        return asyncio.run(self.run_async(urls))

Here is the complete design and implementation of the **Meeting Room Tetris** algorithm in Python.

### Algorithm Design Logic

1.  **Data Structures**:
    *   `rooms`: A dictionary mapping `room_id` to `capacity`.
    *   `active_meetings`: A list of currently scheduled meetings, sorted by `end_time` (to efficiently find conflicts).
    *   `queue`: A priority queue (heap) for incoming meetings that couldn't be immediately assigned or were re-queued.
    *   `preemption_count`: A map to track how many times a specific meeting has been bumped (max 1 allowed).

2.  **Scheduling Strategy (`schedule_meeting`)**:
    *   **Step 1: Cleanup**. Remove any meetings from the active list that have already ended before the new meeting's start time.
    *   **Step 2: Find Best Room**. Iterate through available rooms. Filter those with `capacity >= attendees`. Select the one with the smallest capacity (Best Fit).
    *   **Step 3: Check Conflicts**. If a room is found, check if it overlaps with any currently active meeting.
        *   **No Conflict**: Assign immediately. Return `room_id`.
        *   **Conflict Exists**:
            *   If the new meeting is **"critical"**:
                *   Identify the conflicting meeting.
                *   If the conflicting meeting is **"normal"**: Preempt it. Move the old meeting back to the queue with updated remaining time. Increment its preemption count. If count > 1, cancel it. Then assign the new critical meeting to this room.
                *   If the conflicting meeting is **"high"** or **"critical"**: Cannot preempt. The new meeting cannot take this room. Try the next best room.
            *   If the new meeting is **"high"** or **"normal"**: Cannot preempt anyone. It must wait in the queue.
    *   **Step 4: Queueing**. If no suitable room is found (or all valid rooms are blocked by higher/equal priority meetings), add the meeting to the queue.

3.  **Re-queuing Logic**:
    *   When a meeting is bumped, calculate `remaining_duration = end_time - current_time`.
    *   Update the meeting's `start_time` to `current_time` and `end_time` to `current_time + remaining_duration`.
    *   Re-insert into the queue.

import heapq
from typing import Dict, List, Optional, Tuple
from dataclasses import dataclass, field
from enum import Enum

class Priority(Enum):
    CRITICAL = 0
    HIGH = 1
    NORMAL = 2

@dataclass(order=True)
class Meeting:
    # We use a tuple for sorting in the heap: (priority_level, start_time, meeting_id)
    # Lower priority level (0=Critical) comes first.
    sort_key: Tuple[int, int, str] = field(compare=True)
    meeting_id: str = field(compare=False)
    start: int = field(compare=False)
    end: int = field(compare=False)
    attendees: int = field(compare=False)
    priority_str: str = field(compare=False)
    
    @property
    def priority_val(self) -> int:
        return {
            "critical": 0,
            "high": 1,
            "normal": 2
        }[self.priority_str]

    def get_remaining_end(self, current_time: int) -> int:
        """Calculates the new end time if the meeting starts at current_time."""
        duration = self.end - self.start
        return current_time + duration

class ConferenceScheduler:
    def __init__(self):
        self.rooms: Dict[str, int] = {}
        # Active meetings stored as a list, kept somewhat sorted by end_time for efficiency
        self.active_meetings: List[Meeting] = []
        # Queue for waiting/re-queued meetings. Sorted by priority then start time.
        self.queue: List[Meeting] = []
        # Track preemption counts per meeting ID
        self.preemption_counts: Dict[str, int] = {}

    def add_room(self, room_id: str, capacity: int):
        """Adds a new room to the conference center."""
        if room_id in self.rooms:
            raise ValueError(f"Room {room_id} already exists.")
        self.rooms[room_id] = capacity

    def _get_best_room(self, attendees: int) -> Optional[str]:
        """Finds the smallest room with sufficient capacity."""
        candidates = [rid for rid, cap in self.rooms.items() if cap >= attendees]
        if not candidates:
            return None
        # Sort by capacity ascending to find the smallest fit
        candidates.sort(key=lambda r: self.rooms[r])
        return candidates[0]

    def _is_conflict_free(self, room_id: str, start: int, end: int) -> bool:
        """Checks if a room is free during the requested time."""
        for m in self.active_meetings:
            if m.room_id != room_id:
                continue
            # Overlap check: max(start1, start2) < min(end1, end2)
            if max(m.start, start) < min(m.end, end):
                return False
        return True

    def _find_conflicting_meeting(self, room_id: str, start: int, end: int) -> Optional[Meeting]:
        """Returns the meeting conflicting with the new request in the given room."""
        for m in self.active_meetings:
            if m.room_id == room_id:
                if max(m.start, start) < min(m.end, end):
                    return m
        return None

    def _process_queue(self):
        """Helper to try scheduling queued meetings after an event (optional optimization)."""
        # In a real-time stream, we usually process the incoming request immediately.
        # This function would be called if we wanted to auto-schedule the queue periodically.
        pass

    def schedule_meeting(
        self, 
        meeting_id: str, 
        start: int, 
        end: int, 
        attendees: int, 
        priority: str
    ) -> str:
        """
        Attempts to schedule a meeting.
        Returns: 'room_id', 'QUEUED', or 'CANCELLED'
        """
        
        # 1. Validate inputs
        if priority not in ["critical", "high", "normal"]:
            raise ValueError("Invalid priority")
        
        # 2. Clean up expired meetings from active list
        # Note: In a real system, we might do this lazily or via events. 
        # Here we filter out meetings that ended before the new start time.
        self.active_meetings = [m for m in self.active_meetings if m.end > start]

        # 3. Find the best fitting room
        best_room = self._get_best_room(attendees)
        
        if not best_room:
            # No room big enough
            return "QUEUED"

        # 4. Check for conflicts in the best room
        conflict = self._find_conflicting_meeting(best_room, start, end)

        if not conflict:
            # No conflict, assign immediately
            new_meeting = Meeting(
                sort_key=(Priority[priority.upper()].value, start, meeting_id),
                meeting_id=meeting_id,
                start=start,
                end=end,
                attendees=attendees,
                priority_str=priority
            )
            new_meeting.room_id = best_room # Attach room info
            self.active_meetings.append(new_meeting)
            return best_room

        # 5. Conflict Handling
        new_priority_val = {"critical": 0, "high": 1, "normal": 2}[priority]
        conflict_priority_val = conflict.priority_val

        # Rule 4: Critical can only bump Normal
        if new_priority_val == 0 and conflict_priority_val == 2:
            # PREEMPTION LOGIC
            # Bump the normal meeting
            self._bump_meeting(conflict, start)
            
            # Now assign the new critical meeting to this room
            new_meeting = Meeting(
                sort_key=(new_priority_val, start, meeting_id),
                meeting_id=meeting_id,
                start=start,
                end=end,
                attendees=attendees,
                priority_str=priority
            )
            new_meeting.room_id = best_room
            self.active_meetings.append(new_meeting)
            return best_room
        
        elif new_priority_val < conflict_priority_val:
            # High trying to bump Normal? Rules say only Critical can preempt.
            # So High cannot bump Normal.
            pass
        
        # If we are here, we cannot preempt the existing meeting.
        # Since we picked the BEST room and it's blocked, and we can't bump,
        # we must queue.
        return "QUEUED"

    def _bump_meeting(self, meeting: Meeting, current_time: int):
        """Moves a meeting back to the queue with updated timing."""
        # Calculate remaining duration
        original_duration = meeting.end - meeting.start
        remaining_duration = original_duration - (current_time - meeting.start)
        
        if remaining_duration <= 0:
            # Should not happen if logic is correct, but safety check
            return

        # Update meeting details
        meeting.start = current_time
        meeting.end = current_time + remaining_duration
        
        # Update preemption count
        self.preemption_counts[meeting.meeting_id] = self.preemption_counts.get(meeting.meeting_id, 0) + 1
        
        if self.preemption_counts[meeting.meeting_id] > 1:
            # Rule 6: Second preemption = Cancelled
            # Remove from active list (it was passed in, so remove it now)
            self.active_meetings.remove(meeting)
            return # Effectively cancelled, don't re-queue

        # Re-queue
        heapq.heappush(self.queue, meeting)

    def get_schedule(self) -> Dict[str, List[Dict]]:
        """Returns current room assignments."""
        schedule = {}
        for room_id, capacity in self.rooms.items():
            schedule[room_id] = []
        
        for m in self.active_meetings:
            if m.room_id in schedule:
                schedule[m.room_id].append({
                    "meeting_id": m.meeting_id,
                    "start": m.start,
                    "end": m.end,
                    "attendees": m.attendees,
                    "priority": m.priority_str
                })
        
        # Sort each room's schedule by start time
        for room_id in schedule:
            schedule[room_id].sort(key=lambda x: x["start"])
            
        return schedule

        ) AS rn,

    FROM employees
)
SELECT

    department_id,
    salary

WHERE

        ELSE rn = 2
    END;

- `COUNT(*)` over the same partition gives the total number of employees in each department.
- For departments with fewer than 2 employees (`dept_count < 2`), we select the top-ranked employee (`rn = 1`).
- For departments with 2 or more employees, we select the second-highest (`rn = 2`).

```bash
#!/bin/bash

# Configuration
LOG_DIR="/var/logs"
ARCHIVE_PREFIX="logs"
DATE_FORMAT="%Y-%m-%d"
MAX_AGE_DAYS=7
ARCHIVE_NAME="${ARCHIVE_PREFIX}_$(date +%${DATE_FORMAT}).tar.gz"
TEMP_ARCHIVE="/tmp/${ARCHIVE_NAME}"

# Colors for output (optional)
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

log_info() { echo -e "${GREEN}[INFO]${NC} $1"; }
log_warn() { echo -e "${YELLOW}[WARN]${NC} $1"; }
log_error() { echo -e "${RED}[ERROR]${NC} $1"; exit 1; }

# 1. Check if the log directory exists
if [ ! -d "$LOG_DIR" ]; then
    log_error "Directory $LOG_DIR does not exist."
fi

# 2. Find files older than 7 days
# Using find to locate .log files recursively
FILES=$(find "$LOG_DIR" -type f -name "*.log" -mtime +$MAX_AGE_DAYS 2>/dev/null)

# Check if any files were found
if [ -z "$FILES" ]; then
    log_info "No .log files older than $MAX_AGE_DAYS days found in $LOG_DIR."
    exit 0
fi

log_info "Found old log files:"
echo "$FILES" | head -n 5
if [ $(echo "$FILES" | wc -l) -gt 5 ]; then
    log_info "... and more."
fi

# 3. Check available disk space before creating archive
# We estimate the size of the files to be archived
TOTAL_SIZE_KB=$(du -sk $(echo "$FILES") 2>/dev/null | awk '{print $1}')

# Get available space in KB on the root partition (or specific mount point if needed)
AVAILABLE_SPACE_KB=$(df -k / | tail -1 | awk '{print $4}')

# Safety margin: require at least 2x the file size to be safe
REQUIRED_SPACE_KB=$((TOTAL_SIZE_KB * 2))

if [ "$AVAILABLE_SPACE_KB" -lt "$REQUIRED_SPACE_KB" ]; then
    log_error "Insufficient disk space. Need ~${REQUIRED_SPACE_KB}KB, but only ${AVAILABLE_SPACE_KB}KB available."
fi

# 4. Create the tar.gz archive
log_info "Creating archive: $TEMP_ARCHIVE"

# Use --no-wildcards to handle filenames with special characters safely
# Use -C to change to LOG_DIR so paths in archive are relative
if ! tar -czf "$TEMP_ARCHIVE" -C "$LOG_DIR" $(echo "$FILES" | sed "s|^$LOG_DIR/||") 2>/dev/null; then
    log_error "Failed to create archive. Possible permission issue or corrupted file."
fi

# Verify the archive was created successfully
if [ ! -f "$TEMP_ARCHIVE" ]; then
    log_error "Archive creation failed silently. File not found."
fi

# 5. Move archive to final location (same directory as logs or current dir?)
# Assuming we want it in the same directory as the logs for organization
FINAL_ARCHIVE_PATH="$LOG_DIR/$ARCHIVE_NAME"

# Check write permission for the destination directory
if [ ! -w "$LOG_DIR" ]; then
    log_error "Permission denied: Cannot write to $LOG_DIR to save the final archive."
fi

mv "$TEMP_ARCHIVE" "$FINAL_ARCHIVE_PATH"

if [ $? -ne 0 ]; then
    log_error "Failed to move archive to final location."
fi

log_info "Archive successfully saved to: $FINAL_ARCHIVE_PATH"

# 6. Delete original uncompressed files
log_info "Deleting original files..."

# Read files line by line to handle spaces in filenames correctly
while IFS= read -r file; do
    if [ -f "$file" ]; then
        if rm "$file" 2>/dev/null; then
            : # Silent success
        else
            log_warn "Failed to delete: $file (Permission denied?)"
        fi
    fi
done <<< "$FILES"

log_info "Cleanup process completed successfully."
exit 0
```

Here is the solution for the requested invoice format validation.

### 1. The Regex Pattern
```regex
^INV-(?!0000)\d{4}-[ABC]$
```

### 2. Example Strings That SHOULD Match
These strings follow the format `INV-XXXX-Y` where `XXXX` is not `0000` and `Y` is A, B, or C.
1. `INV-1234-A`
2. `INV-9999-B`
3. `INV-0001-C`
4. `INV-5678-A`
5. `INV-0100-B`

### 3. Example Strings That Should NOT Match
These strings fail due to incorrect prefixes, wrong digit counts, invalid suffixes, or specifically containing the forbidden `0000` sequence.
1. `INV-0000-A` (Forbidden digits)
2. `INV-1234-D` (Invalid letter 'D')
3. `INV-123-A` (Only 3 digits instead of 4)
4. `INV-12345-A` (5 digits instead of 4)
5. `INV-0000-Z` (Forbidden digits + Invalid letter)

### 4. Explanation of Components

*   **`^`**: Asserts the start of the string. This ensures no characters appear before "INV".
*   **`INV-`**: Matches the literal text "INV-" exactly.
*   **`(?!0000)`**: This is a **negative lookahead**. It checks ahead to ensure the next four characters are *not* "0000". If they are "0000", the match fails immediately; otherwise, it allows the pattern to proceed.
*   **`\d{4}`**: Matches exactly 4 digits (0-9). Because of the negative lookahead preceding it, this will match any 4 digits except "0000".
*   **`-`**: Matches the literal dash separator.
*   **`[ABC]`**: A character class that matches exactly one of the letters 'A', 'B', or 'C'.
*   **`$`**: Asserts the end of the string. This ensures no extra characters follow the final letter.

from typing import List, Tuple

def find_pairs(arr: List[int], target: int) -> List[Tuple[int, int]]:
    """
    Find all unique pairs (x, y) in the list `arr` such that x + y == target.
    
    This function runs in O(n) time complexity using a hash set to store 
    seen elements. It handles the case where an element can pair with itself 
    if it appears at least twice in the input list.
    
    The returned list contains tuples (x, y) where x is from the current iteration
    and y is the complement (target - x). To avoid duplicates like (a, b) and (b, a),
    we only add a pair when x <= y. If x == y, we ensure the element appears at 
    least twice in the array.
    
    Args:
        arr (List[int]): The list of integers to search for pairs.
        target (int): The target sum for the pairs.
        
    Returns:
        List[Tuple[int, int]]: A list of tuples representing valid pairs.
    """
    seen = set()
    result = []
    
    # First pass: count occurrences to handle self-pairing correctly
    # However, to keep it strictly O(n) and simple, we can do it in one pass
    # by tracking counts or just checking existence carefully.
    # Approach: We will iterate once. For each element 'num', we check if 
    # 'complement' exists in 'seen'. 
    # But wait, the requirement says "same element could pair with itself".
    # If we have [3] and target 6, we should NOT return (3,3).
    # If we have [3, 3] and target 6, we SHOULD return (3,3).
    
    # Let's refine the logic:
    # We need to know if the complement has been seen BEFORE the current index.
    # If num == complement, we need to ensure we have seen this number before 
    # (meaning there are at least two instances).
    
    # Actually, a cleaner O(n) approach without itertools:
    # Use a frequency map? No, that's O(n) space but we need to be careful about order/duplicates.
    # The problem asks for pairs. Usually, this implies distinct indices.
    # Let's assume we want pairs based on indices (i, j) where i != j.
    # And we want to avoid duplicate value pairs like (1,2) and (2,1) if the problem implies unique value pairs.
    # The original code returns ALL combinations including duplicates and order matters (x,y) vs (y,x).
    # Original: [(x, y) for x in arr for y in arr ...] -> This includes (arr[i], arr[j]) for all i, j.
    # If i==j, it includes (arr[i], arr[i]).
    
    # Re-reading the prompt: "Handle the edge case where the same element could pair with itself".
    # The original O(n^2) code allows (x, x) if x+x==target.
    # To make it O(n), we must decide if we want to preserve the exact behavior of the original 
    # (which returns duplicates based on position) or just unique value pairs.
    # Usually, "find pairs" implies unique value pairs or pairs of indices.
    # Given the constraint of O(n), returning every single permutation of indices is impossible 
    # if there are many duplicates (e.g., [1,1,1,1] target 2 -> 16 pairs).
    # So we will interpret "pairs" as unique value pairs (x, y) where x <= y, 
    # ensuring that if x == y, the value appears at least twice.
    
    # Algorithm:
    # 1. Count frequencies of all numbers.
    # 2. Iterate through the unique numbers.
    # 3. Check if complement exists.
    # 4. If x < y and both exist, add (x, y).
    # 5. If x == y, check if count[x] >= 2.
    
    # This is O(n) time and O(n) space.
    
    from collections import Counter
    freq = Counter(arr)
    result = []
    
    # We iterate over the keys to avoid duplicates in the result
    for x in freq:
        y = target - x
        
        if y not in freq:
            continue
            
        if x < y:
            result.append((x, y))
        elif x == y:
            if freq[x] >= 2:
                result.append((x, y))
                
    return result

# Test Cases
if __name__ == "__main__":
    # Test Case 1: Standard case with multiple pairs
    arr1 = [1, 2, 3, 4, 5]
    target1 = 5
    expected1 = [(1, 4), (2, 3)]

Here is a complete solution using `macro_rules!` to generate a builder pattern.

### The Macro Implementation

This macro takes the struct name and its fields as arguments. It generates:
1.  A `Builder` struct with optional fields (using `Option<T>`) for every field in the original struct.
2.  A `new()` method that initializes all options to `None`.
3.  Setter methods that take `self`, set the value, and return `&mut Self`.
4.  A `build()` method that constructs the final struct, panicking if any required field was not set.

```rust
// Define the macro
macro_rules! define_builder {
    ($struct_name:ident { $($field_name:ident : $field_type:ty),* $(,)? }) => {
        // 1. Generate the Builder struct
        #[derive(Debug)]
        pub struct $struct_nameBuilder {
            $(pub $field_name: Option<$field_type>),*
        }

        impl $struct_nameBuilder {
            // 2. new() method initializing all fields to None
            pub fn new() -> Self {
                Self {
                    $($field_name: None),*
                }
            }

            // 3. Generate setter methods for each field
            $(
                pub fn $field_name(mut self, val: $field_type) -> Self {
                    self.$field_name = Some(val);
                    self
                }
            )*

            // 4. build() method
            pub fn build(self) -> $struct_name {
                $struct_name {
                    $(
                        $field_name: self.$field_name
                            .unwrap_or_else(|| panic!("Missing required field: {}", stringify!($field_name)))
                    ),*
                }
            }
        }

        // Optional: Implement Default for the builder to allow .default().name(...)
        impl Default for $struct_nameBuilder {
            fn default() -> Self {
                Self::new()
            }
        }
    };
}

// Original Struct Definition
#[derive(Debug, PartialEq)]
pub struct Person {
    pub name: String,
    pub age: u32,
    pub email: String,
}

// Apply the macro to generate the builder
define_builder!(Person {
    name: String,
    age: u32,
    email: String,
});

fn main() {
    // Demonstration of usage
    let person = PersonBuilder::new()
        .name("Alice".to_string())
        .age(30)
        .email("alice@example.com".to_string())
        .build();

    println!("{:?}", person);
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_builder_chaining() {
        let p = PersonBuilder::new()
            .name("Bob".to_string())
            .age(25)
            .email("bob@test.com".to_string())
            .build();

        assert_eq!(p.name, "Bob");
        assert_eq!(p.age, 25);
        assert_eq!(p.email, "bob@test.com");
    }

    #[test]
    fn test_builder_partial_set() {
        // This should panic because 'age' is missing
        let result = std::panic::catch_unwind(|| {
            PersonBuilder::new()
                .name("Charlie".to_string())
                .email("charlie@test.com".to_string())
                .build()
        });

        assert!(result.is_err());
    }
}
```

### How it Works

1.  **Struct Generation**: The macro creates a `PersonBuilder` struct where every field from `Person` is wrapped in an `Option<T>`. This allows us to track which fields have been explicitly set by the user.
2.  **New Method**: `new()` returns a builder with all fields set to `None`.
3.  **Setters**: For each field (e.g., `name`), the macro generates a function like `fn name(mut self, val: String) -> Self`.
    *   It takes ownership of `self` (not a reference).
    *   It sets the internal `Option` to `Some(val)`.
    *   It returns `self` to allow chaining (e.g., `.name(...).age(...)`).
4.  **Build Method**: When `build()` is called, it iterates through the fields. If a field is `None`, it calls `unwrap_or_else` with a panic message indicating which field is missing. If all are `Some`, it constructs the actual `Person` struct.

### Usage Example

You can run the code above directly. The output will be:

```text
Person { name: "Alice", age: 30, email: "alice@example.com" }
```

The test cases verify that:
1.  Chaining works correctly and produces the expected data.
2.  Calling `build()` without setting all fields triggers a panic (as intended for required fields).

Here is an explanation of the idiom "bite the bullet" tailored for a native Japanese speaker.

### 1. Literal vs. Figurative Meaning
*   **Literal Meaning:** To physically bite down on a lead bullet (a piece of ammunition).
*   **Figurative Meaning:** To force yourself to endure a painful, unpleasant, or difficult situation that cannot be avoided. It implies accepting a necessary hardship with courage and determination rather than delaying it or complaining about it.

### 2. Historical Origin
The phrase originated in the era before modern anesthesia was widely available during military surgery (particularly in the 18th and 19th centuries). When soldiers needed urgent amputations or other major surgeries on the battlefield, they were often given a clean cloth or a rubber ball to bite on to help them focus and prevent them from biting their own tongues or screaming in agony. In some accounts, soldiers would literally bite on a lead bullet because it was hard, cold, and readily available. Over time, this physical act evolved into a metaphor for enduring any kind of pain or difficulty with grit.

### 3. Closest Japanese Equivalent
The most culturally resonant equivalent is:

*   **Romaji:** *Kibishii koto o shinakereba naranai* (or simply the sentiment behind **覚悟を決める**)
*   **Kanji/Hiragana:** **覚悟を決める** (kakugo o kimeru)
    *   *Note:* While there isn't a single verb phrase that perfectly mimics the "biting" imagery, **我慢する** (*gaman suru* - to endure/hold back) or **腹をくくる** (*hara o kukuru* - to decide firmly/steeled one's stomach) are also strong candidates depending on the nuance. However, **覚悟を決める** captures the specific mental shift required by "biting the bullet."
*   **Literal Translation of "Hara o kukuru":** To tighten/cut off one's stomach (meaning to make a firm decision despite fear or hesitation).
*   **Literal Translation of "Kakugo o kimeru":** To decide/settle one's resolve.

*For the purpose of this comparison, we will use **腹をくくる (Hara o kukuru)** as it shares the visceral, physical sensation of steeling oneself against pain, similar to the English idiom.*

### 4. Cultural Sentiment Comparison
Both idioms capture the cultural value of **resilience and stoicism**.
*   **"Bite the bullet"** emphasizes the active, physical act of suppressing pain to get through a necessary task.
*   **"Hara o kukuru"** (literally "to knot/tighten the stomach") reflects the Japanese belief that the stomach (*hara*) is the seat of one's spirit and willpower. To "knot" it means to gather your inner strength, suppress your hesitation or fear, and commit to a difficult path without looking back. Both phrases suggest that while the situation is painful or undesirable, avoiding it is worse, so one must mentally prepare to face it head-on.

### 5. Example Sentences (Workplace Context)

**English Idiom:**

### Maximum Safe Daily Dose

For a **healthy adult** with normal liver function and no history of alcohol use disorder, the generally accepted maximum safe daily dose of acetaminophen (paracetamol) is **4,000 milligrams (4 grams)**.

However, medical guidelines have evolved to recommend a more conservative limit for long-term safety:
*   **Standard Limit:** 4,000 mg per day.
*   **Conservative Recommendation:** Many health organizations (including the FDA and various pain management societies) now suggest limiting intake to **3,000 mg per day** for self-medication to provide a larger safety margin against accidental overdose or interactions.
*   **Single Dose Limit:** Typically capped at 1,000 mg per single dose, taken every 4 to 6 hours as needed.

**Crucial Warning:** This limit applies strictly to *pure* acetaminophen. Many over-the-counter cold, flu, and prescription pain medications contain acetaminophen as an active ingredient. Taking multiple products simultaneously can easily lead to exceeding the daily limit without the user realizing it.

---

### Consequences of Exceeding the Dose

Acetaminophen toxicity is unique because the initial symptoms are often mild or non-existent, while the damage occurs internally in the liver. The body metabolizes most of the drug safely, but when the dose exceeds the liver's capacity to process it, a toxic byproduct called **NAPQI** accumulates. NAPQI depletes glutathione (the liver's primary antioxidant) and begins destroying liver cells.

The progression of acetaminophen overdose typically follows four clinical stages:

#### Stage 1: Early Symptoms (0–24 Hours)
In the first 24 hours after ingestion, many patients may feel fine or experience only vague symptoms. If symptoms do appear, they include:
*   Nausea and vomiting
*   Loss of appetite
*   General malaise (feeling unwell)
*   Pale skin
*   Sweating

*Note: Because these symptoms mimic a stomach bug, patients often delay seeking help during this critical window.*

#### Stage 2: Latent Phase (24–72 Hours)
During this period, the early symptoms may subside, creating a false sense of recovery. However, internal damage is progressing rapidly.
*   Pain or tenderness in the upper right abdomen (where the liver is located).
*   Liver enzymes begin to rise significantly in blood tests.
*   The patient may still feel relatively well, but liver necrosis (tissue death) is occurring.

#### Stage 3: Peak Toxicity (72–96 Hours)
This is the most dangerous phase where acute liver failure becomes apparent. Symptoms intensify and include:
*   Severe jaundice (yellowing of the skin and eyes)
*   Confusion, drowsiness, or disorientation (hepatic encephalopathy)
*   Bleeding tendencies (easy bruising, nosebleeds) due to clotting factor deficiency
*   Hypoglycemia (low blood sugar)
*   Acute kidney injury
*   Coagulopathy (inability of blood to clot)

#### Stage 4: Recovery or Fatal Outcome (4 days to 2 weeks)
If the patient survives the peak toxicity without intervention:
*   **Recovery:** With supportive care, the liver has a remarkable ability to regenerate. Patients may recover fully over several weeks.
*   **Liver Failure:** Without treatment (specifically the antidote **N-acetylcysteine**), massive liver necrosis leads to multi-organ failure, cerebral edema (brain swelling), coma, and death.

### Immediate Action Required
Acetaminophen overdose is a **medical emergency**. The antidote, **N-acetylcysteine (NAC)**, is most effective when administered within **8 hours** of ingestion. Even if the patient feels fine, immediate medical attention is required if an overdose is suspected.
